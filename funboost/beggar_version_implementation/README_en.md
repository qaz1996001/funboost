# This is the Beggar Version Code Implementation

Primarily strips away many features, greatly simplifying the code lines, demonstrating how the framework distributes the execution of Python functions in a distributed manner. Do not use this version directly, as the functionality is too limited.

The complete version supports 5 types of concurrency, while the beggar version only supports multi-threading concurrency.

The complete version supports 20 types of function auxiliary controls, including rate limiting, timeout termination, consumption confirmation and 15 other features. The beggar version, for the sake of simplifying code demonstrations, does not support any of these.

The complete version supports 30 types of message queue middleware, while this version only demonstrates using Redis (which is popular with everyone) as the middleware.
